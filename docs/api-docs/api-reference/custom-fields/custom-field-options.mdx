Custom field options define the selectable values for `enum` and `enum_multi` custom field definitions.

## Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/v2/custom_field_definitions/:id/options` | List options for a definition |
| GET | `/v2/custom_field_definitions/:id/options/:option_id` | Get a single option |
| POST | `/v2/custom_field_definitions/:id/options` | Create an option |
| PATCH | `/v2/custom_field_definitions/:id/options/:option_id` | Update an option |
| DELETE | `/v2/custom_field_definitions/:id/options/:option_id` | Delete an option |

## Option attributes

| Attribute | Type | Required | Description |
|-----------|------|----------|-------------|
| `label` | string | Yes | Display label shown to users. |
| `value` | string | Yes | Stored value (unique per definition). Alphanumeric, spaces, hyphens, and underscores only. |
| `position` | integer | No | Sort order. Auto-incremented if not provided. |

## List options

Returns all options for a custom field definition.

```json http
{
  "method": "get",
  "url": "https://api.terminal49.com/v2/custom_field_definitions/:id/options",
  "headers": {
    "Content-Type": "application/vnd.api+json",
    "Authorization": "Token YOUR_API_KEY"
  }
}
```

### Response

  "data": [
    {
      "id": "opt123",

## Get an option

Retrieve a single option by ID.

```json http
{
  "method": "get",
  "url": "https://api.terminal49.com/v2/custom_field_definitions/:id/options/:option_id",
  "headers": {
    "Content-Type": "application/vnd.api+json",
    "Authorization": "Token YOUR_API_KEY"
  }
}
```

## Create an option

Add a new option to an enum or enum_multi definition.

```json http
{
  "method": "post",
  "url": "https://api.terminal49.com/v2/custom_field_definitions/:id/options",
  "headers": {
    "Content-Type": "application/vnd.api+json",
    "Authorization": "Token YOUR_API_KEY"
  },
  "body": {
    "data": {
      "type": "custom_field_option",
      "attributes": {
        "label": "Urgent",
        "value": "urgent",
        "position": 1
      }
    }
  }
}
```

<Note>
Options can only be added to definitions with `data_type` of `enum` or `enum_multi`.
</Note>

### Value constraints

The `value` attribute:
- Must be unique within the definition
- Can only contain alphanumeric characters, spaces, hyphens, and underscores
- Is automatically sanitized on creation

## Update an option

Update an existing option's label or position.

```json http
{
  "method": "patch",
  "url": "https://api.terminal49.com/v2/custom_field_definitions/:id/options/:option_id",
  "headers": {
    "Content-Type": "application/vnd.api+json",
    "Authorization": "Token YOUR_API_KEY"
  },
  "body": {
    "data": {
      "type": "custom_field_option",
      "attributes": {
        "label": "Critical Priority",
        "position": 0
      }
    }
  }
}
```

## Delete an option

Remove an option from a definition.

```json http
{
  "method": "delete",
  "url": "https://api.terminal49.com/v2/custom_field_definitions/:id/options/:option_id",
  "headers": {
    "Content-Type": "application/vnd.api+json",
    "Authorization": "Token YOUR_API_KEY"
  }
}
```

<Warning>
Deleting an option does not automatically update existing custom field values that reference it.
</Warning>

## Example: Setting up a priority field

1. Create the enum definition:

```json http
{
  "method": "post",
  "url": "https://api.terminal49.com/v2/custom_field_definitions",
  "headers": {
    "Content-Type": "application/vnd.api+json",
    "Authorization": "Token YOUR_API_KEY"
  },
  "body": {
    "data": {
      "type": "custom_field_definition",
      "attributes": {
        "entity_type": "Shipment",
        "api_slug": "priority",
        "display_name": "Priority",
        "data_type": "enum"
      }
    }
  }
}
```

2. Add the options (repeat for each option):

```json http
{
  "method": "post",
  "url": "https://api.terminal49.com/v2/custom_field_definitions/:id/options",
  "headers": {
    "Content-Type": "application/vnd.api+json",
    "Authorization": "Token YOUR_API_KEY"
  },
  "body": {
    "data": {
      "type": "custom_field_option",
      "attributes": {
        "label": "High",
        "value": "high",
        "position": 1
      }
    }
  }
}
```

3. Now you can set custom field values using the option values. See [Custom fields](/api-docs/api-reference/custom-fields/custom-fields) for details.
