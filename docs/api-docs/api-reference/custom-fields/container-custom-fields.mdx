---
title: Container custom fields
description: Manage custom fields scoped to a specific container.
---

Use these endpoints to manage custom fields for a specific container. These endpoints use `api_slug` instead of ID for updates and deletes.

## List container custom fields

<ParamField path="GET" type="/api/v2/containers/:container_id/custom_fields">
  Returns all custom fields for a container.
</ParamField>

### Example request

```bash
curl -X GET "https://api.terminal49.com/v2/containers/b2fc728c-e2f5-4a99-8899-eb7b34ef22d7/custom_fields" \
  -H "Authorization: Token YOUR_API_TOKEN" \
  -H "Content-Type: application/vnd.api+json"
```

## Create or update a container custom field

<ParamField path="POST" type="/api/v2/containers/:container_id/custom_fields">
  Creates a new custom field or updates an existing one if it already exists for the given `api_slug`.
</ParamField>

### Request body attributes

<ParamField body="api_slug" type="string" required>
  The `api_slug` of the custom field definition.
</ParamField>

<ParamField body="value" type="any" required>
  The field value.
</ParamField>

<ParamField body="update_pathway" type="string">
  Source of the update. Values: `api`, `dashboard`, `email`, `tracking_request`, `unknown`. Defaults to `api`.
</ParamField>

### Example request

```bash
curl -X POST "https://api.terminal49.com/v2/containers/b2fc728c-e2f5-4a99-8899-eb7b34ef22d7/custom_fields" \
  -H "Authorization: Token YOUR_API_TOKEN" \
  -H "Content-Type: application/vnd.api+json" \
  -d '{
    "data": {
      "type": "custom_field",
      "attributes": {
        "api_slug": "cargo_weight",
        "value": 52000.75
      }
    }
  }'
```

### Example response

```json
{
  "data": {
    "id": "cf-3456-7890-12cd-ef3456789012",
    "type": "custom_field",
    "attributes": {
      "value": 52000.75,
      "display_value": "52,000.75",
      "update_pathway": "api",
      "created_at": "2024-01-15T12:00:00Z",
      "updated_at": "2024-01-15T12:00:00Z"
    },
    "relationships": {
      "entity": {
        "data": {
          "id": "b2fc728c-e2f5-4a99-8899-eb7b34ef22d7",
          "type": "container"
        }
      },
      "definition": {
        "data": {
          "id": "def-uuid",
          "type": "custom_field_definition"
        }
      }
    }
  }
}
```

## Update a container custom field by slug

<ParamField path="PATCH" type="/api/v2/containers/:container_id/custom_fields/:api_slug">
  Updates a custom field using its `api_slug`.
</ParamField>

### Request body attributes

<ParamField body="value" type="any" required>
  The new field value.
</ParamField>

<ParamField body="update_pathway" type="string">
  Source of the update.
</ParamField>

### Example request

```bash
curl -X PATCH "https://api.terminal49.com/v2/containers/b2fc728c-e2f5-4a99-8899-eb7b34ef22d7/custom_fields/cargo_weight" \
  -H "Authorization: Token YOUR_API_TOKEN" \
  -H "Content-Type: application/vnd.api+json" \
  -d '{
    "data": {
      "type": "custom_field",
      "attributes": {
        "value": 53500.00
      }
    }
  }'
```

## Delete a container custom field by slug

<ParamField path="DELETE" type="/api/v2/containers/:container_id/custom_fields/:api_slug">
  Soft deletes a custom field using its `api_slug`.
</ParamField>

### Example request

```bash
curl -X DELETE "https://api.terminal49.com/v2/containers/b2fc728c-e2f5-4a99-8899-eb7b34ef22d7/custom_fields/cargo_weight" \
  -H "Authorization: Token YOUR_API_TOKEN"
```

<Note>
The POST endpoint is idempotentâ€”if a custom field with the given `api_slug` already exists for the container, it will be updated instead of creating a duplicate.
</Note>
