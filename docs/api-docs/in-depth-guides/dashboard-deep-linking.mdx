---
title: Dashboard deep linking
description: Link directly to shipments and containers in the Terminal49 dashboard using reference numbers, container numbers, or bill of lading numbers — no Terminal49 IDs required.
---

You can link directly into the Terminal49 dashboard from external systems like a transportation management system (TMS), ERP, or internal tools. Deep links let your team jump straight to a shipment or container without searching manually.

The key benefit: you don't need to know the Terminal49 internal ID. You can use identifiers you already have, such as a container number, bill of lading number, or any reference number tied to a shipment or container in Terminal49.

## Prerequisites

- A Terminal49 account with active shipments or containers
- Reference numbers, container numbers, or bill of lading numbers associated with your tracked shipments in Terminal49

<Tip>
  Reference numbers are added to shipments and containers when you create a tracking request with `ref_numbers`, or by editing a [shipment](/api-docs/api-reference/shipments/edit-a-shipment) or [container](/api-docs/api-reference/containers/edit-a-container) after creation.
</Tip>

## URL format

Deep links follow this pattern:

```
https://app.terminal49.com/search?q={identifier}
```

Replace `{identifier}` with any of the following:

| Identifier type | Example | URL |
| --- | --- | --- |
| Container number | `TCLU6718159` | `https://app.terminal49.com/search?q=TCLU6718159` |
| Bill of lading number | `MEDUFR030802` | `https://app.terminal49.com/search?q=MEDUFR030802` |
| Booking number | `BKG12345678` | `https://app.terminal49.com/search?q=BKG12345678` |
| Reference number | `PO-2024-00123` | `https://app.terminal49.com/search?q=PO-2024-00123` |

## Use cases

### Link from a TMS or ERP

If your TMS or ERP stores container numbers or purchase order numbers, you can construct a deep link to Terminal49 for each record. This gives your logistics team one-click access to real-time tracking data.

For example, if your ERP has a container number field, build the link as:

```
https://app.terminal49.com/search?q={container_number}
```

### Link from a spreadsheet or report

Add a column to your shipment spreadsheet that generates a Terminal49 link using a reference number:

```
=HYPERLINK("https://app.terminal49.com/search?q=" & A2, "View in Terminal49")
```

### Link from automated notifications

Include deep links in email alerts or Slack notifications so recipients can jump directly to the relevant shipment or container:

```
Shipment PO-2024-00123 has arrived at port.
View details: https://app.terminal49.com/search?q=PO-2024-00123
```

## Supported identifiers

You can deep link using any identifier that is associated with a shipment or container in your Terminal49 account:

- **Container number** — the standard ISO container number (e.g., `TCLU6718159`)
- **Master bill of lading** — the original BOL number used to create the tracking request
- **Booking number** — the carrier booking reference
- **Reference numbers** — any custom reference numbers you've added to the shipment or container (e.g., purchase order numbers, house bill of lading numbers, internal IDs)

<Note>
  The search only matches shipments and containers tracked in your Terminal49 account. If a number isn't found, verify that the shipment is actively tracked and that the reference number has been added.
</Note>

## Adding reference numbers

To make deep linking work with your internal identifiers, make sure your reference numbers are attached to shipments or containers in Terminal49. You can add them in three ways:

### When creating a tracking request

Include `ref_numbers` in the tracking request payload:

```json
{
  "data": {
    "type": "tracking_request",
    "attributes": {
      "request_type": "bill_of_lading",
      "request_number": "MEDUFR030802",
      "scac": "MSCU",
      "ref_numbers": ["PO-2024-00123", "HBL-5678"]
    }
  }
}
```

See [Create a tracking request](/api-docs/api-reference/tracking-requests/create-a-tracking-request) for full details.

### By editing an existing shipment

Update the `ref_numbers` on a shipment using the API:

```bash
curl -X PATCH https://api.terminal49.com/v2/shipments/{shipment_id} \
  -H "Content-Type: application/vnd.api+json" \
  -H "Authorization: Token YOUR_API_KEY" \
  -d '{
    "data": {
      "type": "shipment",
      "attributes": {
        "ref_numbers": ["PO-2024-00123", "HBL-5678"]
      }
    }
  }'
```

See [Edit a shipment](/api-docs/api-reference/shipments/edit-a-shipment) for full details.

### By editing an existing container

Update the `ref_numbers` on a container using the API:

```bash
curl -X PATCH https://api.terminal49.com/v2/containers/{container_id} \
  -H "Content-Type: application/vnd.api+json" \
  -H "Authorization: Token YOUR_API_KEY" \
  -d '{
    "data": {
      "type": "container",
      "attributes": {
        "ref_numbers": ["PO-2024-00123"]
      }
    }
  }'
```

See [Edit a container](/api-docs/api-reference/containers/edit-a-container) for full details.
